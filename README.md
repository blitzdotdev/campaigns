# blitzdotdev/campaigns

Hosted skill bundles that let an AI agent (Claude, Cursor, Codex, etc.) stand up
a fully-configured "campaign" — a Blitz-backed app + the skill that drives it —
from a single sentence like *"help me start a new blitz.dev reddit marketing campaign."*

Each campaign is a directory under `campaigns/` containing:

- `SKILL.md` — the canonical skill file the agent installs and follows.
  Frontmatter (`name`, `description`, `metadata`) is required so harnesses
  (Claude Code, Cursor, etc.) can register the skill once it lands in
  `~/.claude/skills/<name>/`.
- *(optional)* `POINTER.md` — short discovery snippet for always-on rules
  harnesses (Windsurf, etc.).
- *(optional)* `config-template.json` — placeholder config the setup flow
  fills in.

The repo's `manifest.json` (auto-generated by `scripts/build-manifest.mjs`)
is the source of truth `blitz.dev` reads when serving:

- `blitz.dev/campaigns` — agent-readable list (markdown + JSON via `Accept`)
- `blitz.dev/campaigns/<name>/skill.md` — full SKILL.md for one campaign
- `blitz.dev/campaigns/<name>/install.sh` — installer that drops SKILL.md
  into `~/.claude/skills/<name>/` and any sibling agent-harness paths

The campaigns list is also injected into `blitz.dev/skill.md` and
`blitz.dev/agents.md` at runtime so a fresh agent reading those discovers what
campaigns exist without an extra hop.

## Contributing a campaign

1. Fork this repo.
2. Add a new directory: `campaigns/<your-campaign-name>/SKILL.md`.
   - Name should be a short kebab-case slug (`reddit-monitor`, `landing-page-ab`,
     etc.). The full skill name as the agent sees it is `blitz-<name>-campaign`.
3. Frontmatter must include:
   ```yaml
   ---
   name: blitz-<your-campaign>-campaign
   description: >-
     One-paragraph pitch. Should mention the canonical trigger phrases so an
     agent reading SKILL listings can route on intent.
   metadata:
     config_path: ~/.claude/skills/blitz-<your-campaign>-campaign/config.json
     depends_on: <comma-separated sibling skills the campaign needs>
   ---
   ```
4. Update `manifest.json` (or run `scripts/build-manifest.mjs` once we wire it up)
   to add an entry with `name`, `title`, `description`, `trigger_phrases`,
   `skill_path`, `depends_on`, `fork_template_slug` (the Blitz project slug the
   setup flow forks from).
5. Open a PR. Once merged on `main`, `blitz.dev` picks it up within the cache
   TTL (5 min).
